## Why

Single diagnosis reports explain one attempt, but students and teachers need a longitudinal view that shows recurring learning gaps across multiple reports. A per-subject student profile will turn scattered wrong-question diagnoses into evidence-backed priorities for study and teacher intervention.

## What Changes

- Derive student id, student name, test/paper name, and subject from the existing S3 image prefix; do not add new diagnosis request fields for these values.
- Use the existing `recipient_email` as the student email column in metadata while keeping parsed student id as a separate column.
- Store each successful structured diagnosis as a JSON report artifact beside the PDF artifact in S3.
- Persist report-level metadata, including parsed student id, recipient email, student name, subject, test/paper name, diagnosis date, diagnosis report id, JSON report S3 path, PDF artifact S3 path when available, and number of questions analysed.
- Add a requested, per-subject student profile report generated by a separate profile analysis agent from multiple JSON diagnosis reports rather than automatically after every invocation.
- Use semantic gap analysis to identify same or related concept gaps and wrong approaches across diagnosis reports, while preserving evidence back to source reports and questions.
- Identify recurring gaps or broader patterns only when supported by evidence from at least two separate diagnosis reports.
- Produce a written longitudinal report for both student and teacher use, including student study priorities, teacher intervention notes, chapter/topic weakness mapping, recurring versus isolated gaps, and evidence references.
- Avoid cross-subject profile synthesis for this change; each requested profile report is scoped to one subject.

## Capabilities

### New Capabilities

- `student-longitudinal-profile`: Requested per-subject profile reports generated from student diagnosis metadata and structured JSON diagnosis reports, including semantic gap analysis, recurring gap analysis, chapter/topic/concept grouping, student-facing priorities, teacher-facing intervention notes, and evidence grounding.

### Modified Capabilities

- `tutor-invocation`: Parse student/test/subject metadata from existing S3 image prefixes, write structured JSON diagnosis artifacts beside PDF artifacts, and persist report-level metadata for future profile analysis without changing the existing request fields or single-attempt diagnosis analysis contract.

## Impact

- S3 image prefix parsing for student id, student name, test/paper name, and subject metadata.
- Successful diagnosis workflow output handling so structured diagnosis JSON can be written as a durable artifact.
- New metadata storage for student diagnosis report records and JSON artifact references.
- New profile analysis agent/use case and response shape.
- Tests for metadata capture, JSON artifact writing, semantic clustering validation, recurrence thresholds, report generation, and privacy.
